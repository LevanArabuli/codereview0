    - Test: discoverRelatedFiles deduplicates files found by both import and naming patterns, keeping higher-priority reason

    - Test: gatherQuickContext respects maxFiles cap (5)

    - Test: gatherQuickContext respects total size budget (200,000 chars)
    - Test: gatherQuickContext returns empty relatedFiles array when no related files found (does not throw)
    - Test: buildExplorationGuidance generates per-file category entries for each changed file
    - Test: buildExplorationGuidance includes callers, tests, type-definitions categories
  </behavior>
  <action>
    Create `src/context.ts` as a new module. Follow the existing codebase convention of ESM-only imports and accepting Octokit as a parameter.

    **Constants:**
    - `MAX_RELATED_FILES = 5` -- per user decision
    - `MAX_FILE_SIZE = 50_000` -- per-file character limit (Claude's discretion: ~12,500 tokens)
    - `MAX_TOTAL_SIZE = 200_000` -- total character budget (Claude's discretion: ~50,000 tokens)
    - `CODE_EXTENSIONS = ['.ts', '.tsx', '.js', '.jsx']` -- code files only per user decision

    **Import parsing (regex, per user decision -- no AST dependency):**
    - `extractRelativeImports(source: string): string[]` -- regex matching `import ... from './path'`, `import './path'`, `require('./path')`. Only relative paths (starting with ./ or ../). Deduplicate results.
    - Regex: `/(?:import\s+(?:[\s\S]*?\s+from\s+)?|require\s*\(\s*)['"](\.[^'"]+)['"]/g`

    **Import resolution:**
    - `resolveImportPath(importingFile: string, importPath: string): string[]` -- uses `path.dirname` + `path.join`. If import already has extension, return as-is. Otherwise, generate candidates: try CODE_EXTENSIONS, then /index + CODE_EXTENSIONS.
    - Normalize paths: strip leading `./` if present for consistency, use forward slashes.

    **Naming pattern inference:**
    - `inferRelatedByNaming(filePath: string): { path: string; reason: 'test' | 'barrel' }[]` -- for each changed file, generate test file candidates (foo.test.ts, foo.spec.ts in same dir AND in tests/ directory if file is in src/), plus barrel file (index.ts in same dir).

    **Discovery orchestration:**
    - `discoverRelatedFiles(changedFiles: PRFile[], changedFileContents: Map<string, string>): { path: string; reason: 'import' | 'test' | 'type' | 'barrel' }[]`
    - For each changed file with content: extract imports, resolve paths, infer naming patterns
    - Filter out files already in the diff (compare against changedFiles filenames)
    - Filter to code files only (CODE_EXTENSIONS check)
    - Deduplicate by path, keeping the higher-priority reason: import > test > type > barrel
    - Sort by priority: imports first, then tests, then types, then barrels

    **Fetching orchestration:**
    - `gatherQuickContext(octokit, owner, repo, ref, changedFiles, changedFileContents): Promise<ReviewContext>`
    - Calls discoverRelatedFiles to get candidates
    - Takes top MAX_RELATED_FILES candidates
    - Fetches in parallel with `Promise.allSettled` using fetchFileContent from github.ts
    - Applies per-file size limit: skip (don't truncate) files over MAX_FILE_SIZE
    - Applies total size budget: stop adding files when MAX_TOTAL_SIZE would be exceeded
    - Returns `{ relatedFiles: [...] }` -- empty array if nothing found (never throws)

    **Deep mode guidance:**
    - `buildExplorationGuidance(changedFiles: PRFile[]): ReviewContext`
    - For each changed file, create an ExplorationCategory with categories: ['callers', 'tests', 'type-definitions']
    - Returns `{ explorationGuidance: [...] }`
    - Claude finds actual files itself per user decision -- we just provide category guidance, not file lists

    **Testing (tests/context.test.ts):**
    - Create comprehensive test file covering all behaviors listed above
    - Use inline test data (no fixtures needed -- pure functions with string inputs)
    - Mock fetchFileContent for gatherQuickContext tests using vi.mock
    - Test budget enforcement edge cases: exactly at limit, one byte over, multiple large files
