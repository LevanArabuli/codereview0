**Commander flag:**
Add `--no-team` option to the Commander program definition. Place it after `--html` and before `--model`:
```typescript
.option('--no-team', 'Run single-agent review instead of parallel team review')
```
This uses Commander's negatable option pattern: `options.team` defaults to `true`, `--no-team` sets it to `false`.

Update the action handler's options type to include `team?: boolean`.

**CLI routing logic:**
Restructure the action handler to route through orchestrator when `options.team !== false`:

For **quick mode** (the `else` branch, when `!options.deep`):
1. If `options.team !== false` (DEFAULT):
   - Print `pc.dim('Running team review...')`
   - Call `analyzeTeamQuick(prData, options.model, options.mode)`
   - Handle the `TeamResult`:
     - Print per-aspect status: for each aspect in ASPECT_TYPES, print `pc.dim('  {Aspect}: ') + (status === 'done' ? pc.green('done') : pc.yellow('failed'))`. Capitalize the aspect name.
     - If `teamResult.allFailed`:
       - Print `pc.yellow('All aspect agents failed -- falling back to single-agent review')`
       - Run existing single-agent `analyzeDiff(prData, options.model, options.mode)` as fallback (ONE attempt, no retry loop). NOTE: Call `analyzeDiff` WITHOUT an aspect argument -- this uses default generalist mode (backwards-compatible, same as original behavior).
       - If fallback also fails, print error and `process.exit(EXIT_ANALYSIS_ERROR)`
     - Otherwise:
       - Print any failed aspects as warnings: `pc.yellow('Warning: {Aspect} aspect failed -- partial results')`
       - Use `teamResult.findings` as findings
       - If `options.verbose`: `printDebug(\`Findings: ${teamResult.rawCount} raw, ${teamResult.findings.length} after dedup\`)`
   - Print model via `printModel(teamResult.model)`
2. If `options.team === false` (`--no-team`):
   - Print `pc.dim('Single-agent mode')`
   - Use existing single-agent path (current code, no changes needed). NOTE: This calls `analyzeDiff(prData, options.model, options.mode)` WITHOUT an aspect argument -- uses default generalist mode.

For **deep mode** (the `if (options.deep)` branch):
Same pattern but with `analyzeTeamDeep`:
1. If `options.team !== false` (DEFAULT):
   - After successful clone, call `analyzeTeamDeep(prData, clonePath, options.model, options.mode, options.verbose)` instead of `analyzeAgentic`
   - Same per-aspect status display
   - Same all-fail fallback (fall back to single-agent `analyzeAgentic`). NOTE: Call `analyzeAgentic` WITHOUT an aspect argument -- uses default generalist mode.
   - Same verbose dedup stats
2. If `options.team === false` (`--no-team`):
   - Print `pc.dim('Single-agent mode')`
   - Use existing single-agent `analyzeAgentic` path (current code). NOTE: Call `analyzeAgentic` WITHOUT an aspect argument -- uses default generalist mode.

**Imports to add:**

**Error handling:**
- Wrap team analysis in try/catch. If the orchestrator itself throws (not individual aspects -- those are handled by Promise.allSettled), catch and fall back to single-agent.
- All error messages pass through `sanitizeError()` (existing pattern).
- Exit code 0 when partial failure (some aspects succeeded). Only `process.exit(EXIT_ANALYSIS_ERROR)` when both team AND fallback fail.

**Per-aspect status format (from CONTEXT.md decision):**
```
  Security: done
  Performance: done
  Quality: failed (timeout)
  Tests: done
```
Use capitalized aspect names. Color: green for done, yellow for failed.

**Key constraints from CONTEXT.md decisions:**
- Team mode is DEFAULT (not opt-in)
- `--no-team` prints "Single-agent mode" confirmation message
- Exit code 0 with warnings when some aspects fail but others succeed
- If ALL 4 fail, auto-fallback to single-agent generalist review
- Same timeout per aspect as current single-agent (already handled in analyzer.ts)
- Verbose shows dedup stats

**Backwards-compatibility note:** All single-agent fallback paths and `--no-team` paths call `analyzeDiff`/`analyzeAgentic` WITHOUT an `aspect` argument. Since Plan 01-02 made the aspect parameter optional with backwards-compatible behavior (buildAspectPrompt without aspect returns the same as buildPrompt), this correctly uses generalist mode.
