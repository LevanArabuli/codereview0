<objective>
Add PR intent extraction and intent-based flagging guidance to the review pipeline.

Purpose: Reviews calibrated against PR intent reduce noise (e.g., a refactor PR does not get flagged for "add tests for new behavior" since there is no new behavior). Intent guides what gets flagged, NOT severity labels -- a bug is always a bug regardless of intent.

Output: `extractIntent()` function, `getIntentGuidance()` prompt builder, `ReviewContext.intent` field, intent threaded through all CLI code paths, comprehensive tests.
</objective>
