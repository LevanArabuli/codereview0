## Directory Purposes

**`src/`:**
- Purpose: All production TypeScript source code
- Contains: 17 modules organized by responsibility (CLI, API, analysis, output)
- Key files: `cli.ts` (entry point), `analyzer.ts` (core engine), `github.ts` (API)

**`tests/`:**
- Purpose: Vitest test suites with fixtures
- Contains: 11 test files covering CLI, analysis, GitHub integration, security
- Key files: `security.test.ts` (39 tests for security invariants), `eval.test.ts` (quality evaluation)

**`tests/fixtures/`:**
- Purpose: JSON test data for deterministic testing
- Contains: Sample PR metadata, diffs, Claude CLI responses
- Key files: Generated by test fixtures, used by eval.test.ts

**`dist/`:**
- Purpose: Compiled JavaScript output
- Contains: Built modules for Node execution
- Generated by: `npm run build` (tsup bundler)

**`.planning/codebase/`:**
- Purpose: GSD architecture documentation
- Contains: ARCHITECTURE.md, STRUCTURE.md (this file)
- Committed: Yes (part of codebase documentation)

**`.codereview/`:**
- Purpose: Runtime clone staging directory for deep reviews
- Contains: Cloned repository checkouts (temporary)
- Generated by: `cloner.ts` during deep mode
- Committed: No (.gitignore)

**`.claude/commands/`:**
- Purpose: GSD command definitions for Claude integration
- Contains: Command metadata and workflows
- Used by: Claude Code IDE

## Key File Locations

**Entry Points:**

- `src/cli.ts`: Main command handler (Commander setup, orchestration, lifecycle)
- `dist/cli.js`: Compiled entry point (built by tsup, referenced in package.json bin field)

**Configuration:**

- `package.json`: NPM metadata, scripts (build/test/lint/dev), dependencies, bin entry
- `tsconfig.json`: TypeScript settings (strict mode, ES2022 target, ESM module)
- `tsup.config.ts`: Bundler config for production build
- `vitest.config.ts`: Test runner config
- `CLAUDE.md`: Project conventions & security invariants (required reading)
- `SECURITY.md`: Threat model, mitigations, API audit, accepted risks

**Core Logic:**

- `src/analyzer.ts`: Claude CLI invocation, subprocess management, response parsing
- `src/github.ts`: Octokit client, PR data fetching, review posting
- `src/cloner.ts`: Safe repo cloning, validation, permission hardening
- `src/prompt.ts`: Review prompt construction, mode overlays
- `src/review-builder.ts`: Finding routing for GitHub posting

**Testing & Evaluation:**

- `tests/security.test.ts`: 39 security invariant tests (no subprocess injection, credential scrubbing, etc.)
- `tests/eval.test.ts`: Quality evaluation with JSON fixtures
- `tests/fixtures/`: JSON test data files

## Naming Conventions

**Files:**

- **Module files**: kebab-case (`cli.ts`, `html-report.ts`, `diff-parser.ts`)
- **Test files**: `{module}.test.ts` (e.g., `analyzer.test.ts` tests `analyzer.ts`)
- **Config files**: Explicit names (`tsconfig.json`, `vitest.config.ts`, `package.json`)
- **Fixture files**: JSON with descriptive names (loaded by test code via createRequire)

**Directories:**

- **Feature areas**: `src/` (source), `tests/` (tests), `dist/` (build output)
- **Staging/runtime**: `.codereview/` (clones), `.claude/` (IDE integration)
- **Documentation**: `.planning/codebase/` (architecture docs)

**Functions & Exports:**

- **Action functions**: `buildPrompt()`, `fetchPRData()`, `parseClaudeResponse()`
- **Validation functions**: `validateGitArg()`, `parsePRUrl()`, `checkPrerequisites()`
- **Formatting functions**: `formatInlineComment()`, `escapeHtml()`, `capitalizeSeverity()`
- **Type guards/helpers**: `isLineInDiff()`, `partitionFindings()`, `filterEnv()`

**Types & Interfaces:**

- **Data types**: `PRData`, `PRFile`, `DiffHunk`, `ReviewFinding` (PascalCase)
- **Result types**: `AnalysisResult`, `PrereqFailure` (PascalCase)
- **Union types**: `ReviewMode` (PascalCase)

## Where to Add New Code

**New Feature (e.g., Jira integration, webhook support):**
- Primary code: `src/{feature}.ts` (new module following existing pattern)
- If GitHub-related: Add to `src/github.ts` as separate function
- Tests: `tests/{feature}.test.ts`
- Integration point: Import in `src/cli.ts`, call in appropriate phase

**New Review Mode (e.g., "aggressive"):**
- Update mode enum: `src/prompt.ts` (add to REVIEW_MODES const)
- Add overlay: `src/prompt.ts` MODE_OVERLAYS record
- CLI option: Already generic in `src/cli.ts` (uses mode from options)
- Tests: Add cases to `tests/prompt.test.ts`

**New Finding Type (e.g., performance issue):**
- Update schema: `src/schemas.ts` (ReviewFindingSchema severity enum)
- Update prompt: `src/prompt.ts` (FINDING_FORMAT_INSTRUCTIONS)
- Update formatter: `src/formatter.ts` if special rendering needed
- Tests: Add cases to analyzer and formatter tests

**New Output Format (e.g., SARIF, XML):**
- Create module: `src/{format}-report.ts`
- Implement: Report generation function taking findings + metadata
- Integration: Call from `src/cli.ts` post-analysis phase (similar to HTML report)
- Tests: `tests/{format}-report.test.ts`

**New GitHub Integration (e.g., auto-approve):**
- Add method to: `src/github.ts` (Octokit wrapper)
- Security check: Ensure read-only by design (see SECURITY.md)
- Call from: `src/cli.ts` post-analysis if appropriate flag set
- Tests: `tests/github.test.ts`

**Utilities & Helpers:**

- Shared validation: `src/errors.ts` or create `src/validation.ts`
- Shared formatting: `src/formatter.ts` (already used widely)
- Shared types: `src/types.ts` (central type definitions)

## Special Directories

**`.planning/codebase/`:**
- Purpose: GSD architecture documentation (consumed by `/gsd:plan-phase` and `/gsd:execute-phase`)
- Generated by: `/gsd:map-codebase` agent
- Committed: Yes
- Contents: ARCHITECTURE.md, STRUCTURE.md (or focus-specific docs)

**`.codereview/`:**
- Purpose: Runtime staging directory for cloned repositories
- Generated by: `src/cloner.ts` during deep review
- Committed: No (listed in .gitignore)
- Cleanup: Automatic on completion or SIGINT; user prompted to keep/delete

**`dist/`:**
- Purpose: Compiled JavaScript output for distribution
- Generated by: `npm run build` (tsup bundler)
- Committed: No (listed in .gitignore)
- How built: `src/cli.ts` entry point compiled to `dist/cli.js` with all dependencies bundled

**`tests/fixtures/`:**
- Purpose: JSON test data (sample PRs, diffs, responses)
- Generated by: Manually created or via test setup
- Committed: Yes (part of test suite)
- Loaded by: `createRequire(import.meta.url)` in tests (ESM-compatible JSON import)
